1 绪论

时间复杂度O

f(x) = O(g(x)) 表示的含义是f(x)以g(x)为上界。

设问题规模为n，存在正常数c、n、n0，当 n>n0 的时，任意的 f(n) 符合 0 <= f(n) <= c.g(n)

逻辑结构分类

存储结构分类

顺序存储：随机存取，存储密度大，但删除、插入操作较耗时。

链式存储：删除、插入操作快，但无法随机存取，存储密度小。

索引存储：需要额外的索引表。

哈希存储（散列存储）：根据元素关键词得到存储下标。

2 顺序表

静态链表的优点

顺序表和单链表的差别 就是 顺序存储和链式存储的差别。

静态链表融合了二者的优点，既能随机访问元素，又能快速插入、删除元素。

头指针和头节点的区别

头指针：指向第一个节点存储位置的指针，具有标识作用；无论链表是否为空，头指针都是链表的必要元素。

头结点：放在第一个元素节点之前，便于在第一个元素节点之前进行插入和删除的操作；头结点不是链表的必须元素，不存储任何信息。

判断链表有没有环

从头开始设置两个指针，快指针每次走2步，慢指针每次走1步，如果快指针先碰到尾，则无环，否则两个指针之后一定会重合，则有环。

3 栈、队列和数组

什么是共享栈？

利用栈底位置相对不变的特性，让两个顺序栈共享一个一维数组空间，将两个栈的栈底分别设置在共享空间 的两端，两个栈顶向共享空间的中间延伸。这样能够更有效的利用存储空间，防止上溢。

4 串

KMP

基于 next 数组（最长公共前后缀），让子串和主串的每个字符进行匹配，当出现匹配失败时，如果已匹配相等的序列中有某个后缀正好是子串的前缀，那么可以直接将子串滑动到与这些相等字符对齐的位置，使得主串指针无须回溯。

BF暴力匹配算法的时间复杂度为 O(mn)，而 KMP 的时间复杂度仅为 O(m+n)

5 树

基本概念

满二叉树：除叶子结点外，每个结点的度都为 2 的二叉树

完全二叉树：除了最后一层外，其他层的每个结点的度都为 2 ，且最后一层也只是在最右侧缺少节点

线索二叉树：将二叉树中所有为空的左、右指针指向某种遍历次序下该节点的 直接前驱、直接后继结点，则得到的就是线索二叉树。

  • 优点：用空指针域来存储结点之间前趋和后继关系，既不增加空间，还能加快查找结点的前驱和后继

双亲表示法、孩子表示法、孩子兄弟表示法；

哈夫曼树

哈夫曼树又叫最优二叉树，在含有 n 个带权叶子结点的二叉树中，所有叶子结点的带权路径长度之和最 小的二叉树是哈夫曼树。

构建过程：

1.在 n 个权值中选出两个最小的权值，对应的两个结点组成一个新的二叉树，且新二叉树的根结点的权值为左右孩子权值的和；

2.在原有的 n 个权值中删除那两个最小的权值，同时将新的权值加入到 n–2 个权值的行列中；

3.重复 1 和 2 ，直到所以的结点构建成了一棵二叉树为止，这棵树就是哈夫曼树。

哈夫曼编码：

从根往下走，往左为0，往右为1，由此可以得到一组无公共前缀的01编码，频率越高的字符，编码长度越短。

6 图

存储方法

邻接矩阵法：空间复杂度高，便于判断两点是否相连，不利于查找出度。

邻接表法：空间复杂度低，利于查找出边，不利于判断两点是否相连。

十字链表法：利于查找出边/入边，不利于判断两点是否相连。

最小生成树

Prim算法：

​ 任选一点作为初始点，加入集合；

​ 找到与当前集合距离最近的顶点，将该点和对应的边加入集合；

​ 重复上述过程，直至将所有顶点都加入集合；

Kruskal算法：

​ 对连通图的所有边按权值排序；

​ 取权值最小的边，按权重从小到大选边；

​ 若加入该边后不构成回路，则取之，否则弃之；

​ 重复上述过程，直至边数=顶点数-1为止。

最短路径问题

Dijkstra：

​ 求解单源最短路径的贪心算法，要求边权不为负;

​ O（N^2）堆优化后，复杂度O（N*logN）

Floyd 算法：

​ 求解多源图最短路径的动态规划算法

​ 时间复杂度为 O(N^3)，空间复杂度为 O(N^2)

拓扑排序

扑排序的使用对象是有向无环图（DAG）。

算法描述：从图中选择入度为 0 的顶点输出，并删除该点和所有以它为起点的边，重复该过程，直到图为空。

7 查找

基本概念

二分查找：递归地在相应的半部分查找，O(logn)

分块查找：查找时先用索引表找到位于哪个块，再在块内进行顺序查找。子表间有序，而子表内无序。

二叉搜索树（二叉排序树）：左子树上所有节点的值都小于根节点的值，右子树上所有节点的值都大于根节点的值，它的左右子树也分别为二叉搜索树。

平衡二叉树：是一种特别的二叉排序树，它的左右子树的高度差的绝对值不超过 1

红黑树：红黑树是一种自平衡的二叉搜索树，保证时间复杂度为 O(logn)

B 树：B 树是所有结点的平衡因子均等于 0 的多路平衡查找树

B+ 树：对 B 树的改进，MySQL 种用的就是 B+ 树。叶子结点直接有指针，构成了一个链表，这个链表是有序的，即可以直接通过遍历链表进行 区间查找

哈希表（散列表）：通过哈希函数将查找表中的关键字映射成一个地址来加快查找速度

红黑树的原理 TODO

平衡二叉树为了保持平衡，需要通过频繁的旋转操作来调整树的结构，比较费时，而红黑树进一步放宽 平衡条件，实际效率更高。

哈希表的构造

直接定址法：取关键字的某个线性函数值作为地址，适用于关键字分布均匀的情况

除留余数法：取关键字对某个数取余的值作为地址

数字分析法：当关键字位数很大时，取分布均匀的任意几位作为地址

平方取中法：取关键字的平方的中间几位作为地址

怎么解决哈希冲突？

哈希冲突指的是两个不同的数经过哈希函数计算后得到了同一个地址。

开放定址法：遇到哈希冲突时，去寻找一个新的空闲的哈希地址

链接法：将所有哈希地址相同的记录都链接在同一链表中

再哈希法：构造多个哈希函数，如果发生哈希冲突时就使用另外的哈希函数计算

溢出表法：将哈希表分为基本表和溢出表，将发生冲突的都存放在溢出表中

8 排序

基本概念

冒泡排序：通过遍历所有的相邻的两个数，如果他们逆序了，就交换位置，依次确定数组中的每个位置；

选择排序：通过遍历所有数，找到第 k 小的数，即数组中位置 k 的元素；

插入排序：就像打扑克时边抓牌边理牌一样，先把第1个数当成已排序序列，第2个数就插到合理位置； 再把第1,2个数当成已排序序列，第3个数就插到合理位置；依次类推即可

堆排序

把所有元素按完全二叉树的顺序存储方式存储在一个一维数组中，

如果父结 点的值比子结点大，则是大根堆；如果父结点的值比子结点小，则是小根堆。

排序稳定性

排序算法的稳定性：数组中值相等的元素，在排序后其相对次序若能保持不变，则算法是稳定的

稳定的算法：冒泡、插入、归并

不稳定的算法：选择、快排

外排序

之前的排序都是内排序，即在内存中完成的；

而对于大数据集，内存是一次性存不下的，需要在内存和 磁盘上进行多次数据交换。
