Lean4函数式编程基础与应用

Lean4与定理证明

1 简介

形式验证（Formal Verification）是指使用逻辑和计算方法来验证用精确的数学术语表达的命题。这包括普通的数学定理，以及硬件或软件、网络协议、机械和混合系统中的形式命题。

在实践中，验证数学命题和验证系统的正确性之间很类似：形式验证用数学术语描述硬件和软件系统，在此基础上验证其命题的正确性，这就像定理证明的过程。

二十世纪的逻辑学发展表明，绝大多数传统证明方法可以化为若干基础系统中的一小套公理和规则。有了这种简化，计算机能以两种方式帮助建立命题：

1）自动定理证明（Automated theorem proving）：为定理寻找一个证明

2）交互式定理证明器（Interactive theorem proving）：验证一个所谓的证明是正确的。

Lean旨在融合交互式和自动定理证明，它将自动化工具和方法置于一个支持用户交互和构建完整公理化证明的框架中。它的目标是支持数学推理和复杂系统的推理，并验证这两个领域的命题。

合取引入：表达式And.intro h1 h2 是 p ∧ q 的证明，它使用了 h1 : p 和 h2 : q 的证明。

  1. 析取

左右析取（或）引入规则：表达式 Or.intro_left q hp 从证明 hp : p 建立了 p ∨ q 的证明。

当Or.intro_right 和 Or.intro_left 的第一个参数可以由 Lean 自动推断出来时，可以简化为 Or.inr 和 Or.inl。

析取消去规则：表达式 Or.elim hpq hpr hqr 接受三个论证，hpq : p ∨ q，hpr : p → r 和 hqr : q → r，生成 r 的证明

  1. 否定

否定 ¬p 真正的定义是 p → False

False 只有一个消去规则 False.elim，它表达了一个事实，即矛盾能导出一切（这个规则有时被称为爆炸原理）。

这种从相互矛盾的假设中推导出任意事实的模式，也可以用 absurd 来表示。

  1. 逻辑等价

表达式 Iff.intro h1 h2 从 h1 : p → q 和 h2 : q → p 生成了 p ↔︎ q 的证明。

表达式 Iff.mp h 从 h : p ↔︎ q 生成了 p → q 的证明；表达式 Iff.mpr h 从 h : p ↔︎ q 生成了 q → p 的证明。

3.3 辅助子目标

Lean 提供了一种帮助构造长证明的方法，即 have 结构，它在证明中引入了一个辅助的子目标。

例如：

表达式 have h : p := s; t 相当于 (fun (h : p) => t) s。

换句话说，s 是 p 的证明，t 是假设 h : p 的期望结论的证明。

5 证明策略

除了直接构造命题实例以外，Lean4还为我们提供了另一种证明命题方法，即策略（Tactic）。

一个证明项代表一个数学证明；策略是描述如何建立这样一个证明的命令或指令。

策略告诉 Lean 如何构建证明。它们自然而然地支持增量式的证明书写，在这种写作方式中，你将分解一个证明，并一步步地实现目标。
